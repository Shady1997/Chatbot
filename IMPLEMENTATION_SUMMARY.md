## 📚 Knowledge Base Integration

### How It Works

1. **Scans** `knowledge/` folder for all documents
2. **Extracts Text** from PDF, Markdown, and DOCX files
3. **Creates Index** of all keywords and concepts
4. **Matches** answer keywords against knowledge base
5. **Scores** relevance (0-100%) and correctness

### What Files Support

| Format | Support | Notes |
|--------|---------|-------|
| `.md` | ✅ Full | Auto-detected |
| `.txt` | ✅ Full | Auto-detected |
| `.pdf` | ⚠️ Optional | Install: `npm install pdf-parse` |
| `.docx` | ⚠️ Optional | Install: `npm install docx-parser` |

---

## 🧮 Evaluation Algorithm

### Step 1: Keyword Extraction
- Extracts significant words from questions and answers
- Removes common stop words (the, a, and, etc.)
- Filters words < 3 characters

### Step 2: Knowledge Base Matching
- Compares extracted keywords against knowledge base
- Counts matching keywords
- Calculates relevance percentage

### Step 3: Correctness Assessment
- Checks answer length (minimum 50 characters)
- Checks for error messages
- Combines relevance with content quality

### Step 4: Classification
- Answers ≥ 60% correctness = "✓ Correct"
- Answers < 60% correctness = "✗ Incorrect"

## 🔍 Understanding the Evaluation Scores

### Relevance Score (0-100%)
"How much of the answer matches knowledge base keywords?"
- 80-100%: Strong match - answer covers knowledge base content
- 60-79%: Good match - answer touches on key concepts
- 40-59%: Partial match - answer mentions some relevant topics
- 0-39%: Weak match - answer mostly irrelevant

### Correctness Score (0-100%)
"Overall how correct is this answer?"
- Combines relevance with content quality checks
- Checks: answer length, no error messages, keyword density
- 0.6+ = "Correct" badge, < 0.6 = "Incorrect" badge
